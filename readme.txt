Sous les "Visibility modifiers and subclasses", on a appris toutes les "classes" et "functions" en kotlin sont par defaut, mais on peut utiliser les modifiers pour changer les "visibility" ou 
public: signifie qu'il est visible hors de la class. Tout est publique par defaut, avec les variables et les methodes de classes visible.

internal: ca signifie que ca sera visible dans cette classe ou le fichier source lorsqu'on travaille avec les "functions" 

private:ca signifie qu'il sera visible dans cette "class" ou le fichier source si on travaille avec les "functions"
protected est pareil que "le private" mais il peut etre visible par n'importe quelle sous classes.
